# Contributing to the To-Do List App

First off, thank you for considering contributing to the To-Do List App! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## 🤝 Code of Conduct

By participating in this project, you agree to abide by our Code of Conduct. We expect all contributors to maintain a respectful, welcoming, and inclusive environment.

## 🛠️ How Can I Contribute?

### Reporting Bugs
If you find a bug in the source code, you can help us by submitting an issue to our GitHub Repository. Even better, you can submit a Pull Request with a fix.

### Suggesting Enhancements
If you have a great idea to improve the app, we want to hear it! Please open an issue outlining your proposal. Check existing issues first to avoid duplicates.

### Pull Requests
We actively welcome your pull requests.

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes (`npm test`).
4. Format your code according to our Oxlint configurations.
5. Update the documentation (like `README.md` or `ARCHITECTURE.md`) if necessary.
6. Submit a Pull Request.

## 🌿 Branching Strategy

- `main` is our stable release branch.
- Feature branches should be named `feature/<feature-name>`.
- Bugfix branches should be named `fix/<bug-name>`.

## 💻 Development Setup

1. Clone your fork locally.
2. Run `npm install` to install dependencies.
3. Use `npm run dev` to start both the backend API and the Vite React server concurrently.
4. Run tests frequently using `npm test` to ensure regressions are caught early.

Thank you for contributing!
